通过对比，每个Python函数都返回一些东西，即使它是 None 值，可以通过省略 return 语句或显式地隐式返回。

__ init __ 方法需要返回 None 。 如果 __ init __ 方法使用 yields 或returns 返回 任意表达式，而不是返回 None，就会引发 TypeError 。返回一些计算结果为 None 的表达式不会引发错误， 但被认为是不好的做法。

不符合规范的代码示例

class MyClass(object):
    def __init__(self):
        self.message = 'Hello'
        return self  # 不符合规范的

合规的解决方案

class MyClass(object):
    def __init__(self):
        self.message = 'Hello'
