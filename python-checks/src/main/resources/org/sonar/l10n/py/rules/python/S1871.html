在具有相同实现的相同if结构中具有两个分支是最好的重复代码，并且最坏的是编码错误。如果两个实例确实需要相同的逻辑，那么它们应该合并。

不符合规范的代码示例

if 0 <= a < 10:
    do_first()
    do_second()
elif 10 <= a < 20:
    do_the_other_thing()
elif 20 <= a < 50:
    do_first()         # 不符合规范的; 重复第一个条件
    do_second()

例外

包含单行代码的 if 链中的块将被忽略。

if 0 <= a < 10:
    do_first()
elif 10 <= a < 20:
    do_the_other_thing()
elif 20 <= a < 50:
    do_first()         # 没异常，通常这是为了提高可读性

但是，当所有分支具有相同的单行代码时，此例外就不适用了。

if 0 <= a < 10:
    do_first()
elif 20 <= a < 50:
    do_first()         # 不合规，这可能是故意的，但也可能不是
