让一个具有相同名称的类成员（不区分大小写）作为其封闭类令人困惑。 当你考虑时尤其如此 为类本身命名类实例的常见做法。

最佳实践要求重命名任何与封闭类同名的字段或成员，以更具描述性 它所代表或持有的阶级方面。

不符合规范的代码示例

class Foo:
  foo = ''

  def getFoo(self):
    ...

foo = Foo()
foo.getFoo() # 这会返回什么？

合规解决方案

class Foo:
  name = ''

  def getName(self):
    ...

foo = Foo()
foo.getName()
