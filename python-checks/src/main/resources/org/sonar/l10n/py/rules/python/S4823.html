使用命令行参数是不安全的。 在过去，它导致了以下漏洞：

  • CVE-2018-7281
  • CVE-2018-12326
  • CVE-2011-3198

命令行参数可能像其他用户输入一样危险。 未经事先验证和清除，切勿使用它们。

还要记住，任何用户都可以检索在系统上运行的进程的列表，从而使提供给他们的参数可见。 从而 通过命令行参数传递敏感信息应该被认为是不安全的。

该规则在每次对 sys.argv 的引用, 对 optparse.OptionParser() 的调用或者对 argparse.ArgumentParser()的调用时会报出安全问题。 目的是指导安全代码审查。
