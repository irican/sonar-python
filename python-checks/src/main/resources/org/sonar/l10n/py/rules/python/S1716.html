break 和 continue 是非结构化的控制流语句，这使得代码难以阅读。 此外，当导入在循环外部包含 break 或者 continue 的模块时，更新版本的Python会引发SyntaxError。

因此，不应在循环之外使用这些语句。

不符合规范的代码示例

narg=len(sys.argv)
if narg == 1:
        print('@Usage: input_filename nelements nintervals')
        break

合规的解决方案

if narg == 1:
        print('@Usage: input_filename nelements nintervals')
        sys.exit()
